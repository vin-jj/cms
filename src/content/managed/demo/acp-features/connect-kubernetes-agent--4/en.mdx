# Overview

Installing the QueryPie Agent allows you to use third-party Kubernetes client tools such as kubectl, Lens, and k9s.

# Downloading the QueryPie Agent

<InfoNote>
    The QueryPie Agent supports Mac, Windows, and Linux operating systems.
</InfoNote>

<BlueState>STEP 1</BlueState> After logging into QueryPie, click on your profile in the top right corner and select the `Agent Download` button.

<BlueState>STEP 2</BlueState> Download the installation file suitable for your PC's operating system.

# Running the Agent and Connecting to a Cluster

<BlueState>STEP 1</BlueState> Enter the QueryPie URL in the **QueryPie Host** field and click `Next`.

<BlueState>STEP 2</BlueState> Log in using your credentials in the browser window.

<BlueState>STEP 3</BlueState> Once login is successful, return to the Agent app.

<BlueState>STEP 4</BlueState> Check the list of accessible resources in the Agent app.
- If you have multiple roles, select the desired role after logging into the Agent.
- Click the magnifying glass icon next to each cluster to view detailed policy information applied to that cluster.

<BlueState>STEP 5</BlueState> In the Agent settings, click the `KubeConfig Path` button and configure the Kubeconfig file path.
- **Path Configuration:** The user can specify the file path where the Kubeconfig file will be saved.
- The default path is `$HOME/.kube/querypie-kubeconfig`.

<BlueState>STEP 6</BlueState> Copy the provided command line and execute it in your terminal to declare the KUBECONFIG environment variable.

<InfoNote>
    1. If this is the first time setting the KUBECONFIG environment variable, replace the default "`${KUBECONFIG}`" value with "`${HOME}/.kube/config`".
    ```sh
    export KUBECONFIG="${HOME}/.kube/config:${HOME}/.kube/querypie-kubeconfig"
    ```
    2. If you’ve changed the custom Kubeconfig path provided by QueryPie, update the environment variable with the new path:
    ```sh
    export KUBECONFIG="${KUBECONFIG}:<User-specified path>"
    ```
    3. Use `Copy` to copy the command line, declare it on your client, and switch the context to an accessible cluster:
    ```sh
    export KUBECONFIG="${KUBECONFIG}:${HOME}/.kube/querypie-kubeconfig"
    ```
</InfoNote>
