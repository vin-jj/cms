# Overview

특정 DB 커넥션에 대해 권한이 없는 쿼리 실행을 요청할 수 있습니다. 승인 후, 실행자는 승인된 쿼리를 1회에 한하여 실행하고, 그 결과를 조회할 수 있습니다.

# 쿼리 실행 요청하기

<BlueState>STEP 1</BlueState> 아래 경로를 통해 **SQL Request** 페이지에 진입합니다.
- **Web Editor**에서 권한 없는 쿼리 실행 > `SEND SQL Requests` 버튼 클릭
- Workflow > Submit Request > SQL Request

<BlueState>STEP 2</BlueState> 결재 규칙을 선택하고 승인자를 지정합니다.
- **Approval Rule:** **General > Approval Rules** 메뉴에서 관리자가 생성한 규칙 목록들이 나타납니다.
  - 규칙에 따라 승인 대상이 달라질 수 있으며, 관리자가 승인자를 지정한 결재 규칙인 경우 자동으로 승인자가 지정됩니다.
- **Approver:** 결재 라인에서 `+` 버튼을 누르고 단계별 승인자를 지정합니다.
  - **A single assignee can complete the approval request:** 여러 명 중 한 명의 승인만으로 결재 요청이 승인 처리됩니다.
  - **All assignees must approve this request:** 모든 승인자가 승인하여야 결재 요청이 승인 처리됩니다.

<BlueState>STEP 3</BlueState> 요청 정보를 입력한 후 `Submit` 버튼을 눌러 Request를 요청합니다.
- **Title:** 요청 제목을 입력합니다.
- **Connection:** SQL 실행을 요청할 대상 커넥션을 선택합니다. 권한이 있는 커넥션에 한해 선택 가능합니다.
- **Database:** 선택한 커넥션 내에서 SQL 실행을 요청할 데이터베이스를 선택합니다.
- **Content Type:** 요청할 SQL 구문을 입력할 방식을 선택합니다.
  - **Text:** 쿼리를 최대 9999라인, 1천만자, 또는 5MB까지 작성할 수 있습니다.
  - **File:** `.sql` 파일을 업로드할 수 있습니다.
- **Target Date:** 해당 쿼리가 실행되기를 원하는 일자를 선택합니다. 기본 선택일은 요청일 당일입니다.
- **Due Date:** 승인/실행이 만료되는 일자를 선택합니다. 기본 선택일은 1년 뒤입니다.
- **Reason for Request:** SQL 실행 권한을 요청하는 사유를 입력합니다.

# 요청받은 결재 승인하기

<BlueState>STEP 1</BlueState> 승인자로 지정된 관리자는 **Workflow > Received Requests > To Do** 메뉴로 이동합니다.

<BlueState>STEP 2</BlueState> 결재의 디테일 페이지에서 `Approve` 버튼을 눌러 승인처리합니다.

# 승인 완료된 쿼리 실행하기

<BlueState>STEP 1</BlueState> 실행자는 **Workflow > Executions > To Do** 메뉴로 이동하여 실행 대기중인 Request를 클릭합니다.

<BlueState>STEP 2</BlueState> SQL Request 상세 내용을 확인 후 우측 상단의 `Execute` 버튼을 클릭하여 실행합니다.
- 상세 내용 확인 후, 실행을 취소하기를 원하는 경우 Cancel 버튼을 클릭하여 취소 처리할 수 있습니다.
- 실행이 완료된 결과는 상세 페이지 내의 `Results` 탭에서 성공 / 실패 여부를 확인할 수 있습니다.
- **Workflow > Executions > Done** 메뉴에서 본인이 실행한 요청 내역을 확인할 수 있습니다.
