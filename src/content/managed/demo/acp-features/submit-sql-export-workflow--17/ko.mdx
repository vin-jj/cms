# Overview

아래 순서에 따라 SQL Export Request를 상신할 수 있습니다. 특정 DB 커넥션에 대해 권한이 없는 데이터 내보내기를 요청하게 되며 승인 후, 실행자는 승인받은 데이터 내보내기를 1회에 한해 실행할 수 있습니다.

# 데이터 다운로드 요청하기

<BlueState>STEP 1</BlueState> 아래 경로를 통해 **SQL Export Request** 페이지에 진입합니다.
- Workflow > Submit Request > SQL Export Request

<BlueState>STEP 2</BlueState> 결재 규칙을 선택하고 승인자를 지정합니다.
- **Approval Rule:** **General > Approval Rules** 메뉴에서 관리자가 생성한 규칙 목록들이 나타납니다.
  - 규칙에 따라 승인 대상이 달라질 수 있으며, 관리자가 승인자를 지정한 결재 규칙인 경우 자동으로 승인자가 지정됩니다.
- **Approver:** 결재 라인에서 `+` 버튼을 누르고 단계별 승인자를 지정합니다.
  - **A single assignee can complete the approval request:** 여러 명 중 한 명의 승인만으로 결재 요청이 승인 처리됩니다.
  - **All assignees must approve this request:** 모든 승인자가 승인하여야 결재 요청이 승인 처리됩니다.

<BlueState>STEP 3</BlueState> 요청 정보를 입력한 후 `Submit` 버튼을 눌러 Request를 요청합니다.
- **Title:** 요청 제목을 입력합니다.
- **Connection:** SQL 실행을 요청할 대상 커넥션을 선택합니다. 권한이 있는 커넥션에 한해 선택 가능합니다.
- **Database:** 선택한 커넥션 내에서 데이터 내보내기를 요청할 데이터베이스를 선택합니다.
- **Content Type:** 다운로드하고자 하는 데이터를 조회할 방식을 선택합니다.
  - **Query Result:** 요청 페이지 내의 Queries 항목에서 쿼리를 직접 작성합니다. 해당 쿼리 결과를 다운로드할 수 있습니다. 최대 10개까지 쿼리를 작성하고 Export 가능합니다.
  - **Table:** 선택한 테이블 전체를 내보내기 할 수 있습니다.
- **Target Date:** 해당 쿼리가 실행되기를 원하는 일자를 선택합니다. 기본 선택일은 요청일 당일입니다.
- **Due Date:** 다운로드가 실행되기를 원하는 일자를 선택합니다. 따로 설정하지 않으면 요청일로 기본 선택됩니다.
- **Reason for Request:** SQL Export 권한을 요청하는 사유를 입력합니다.

# 요청받은 결재 승인하기

<BlueState>STEP 1</BlueState> 승인자로 지정된 관리자는 **Workflow > Received Requests > To Do** 메뉴로 이동합니다.

<BlueState>STEP 2</BlueState> 결재의 디테일 페이지에서 `Approve` 버튼을 눌러 승인처리합니다.

# 승인 완료된 데이터 다운로드하기

<BlueState>STEP 1</BlueState> 실행자는 **Workflow > Executions > To Do** 메뉴로 이동하여 실행 대기중인 Request를 클릭합니다.

<BlueState>STEP 2</BlueState> SQL Export Request 상세 내용을 확인 후 우측 상단의 `Execute` 버튼을 클릭하여 실행합니다.

<BlueState>STEP 3</BlueState> Export Option 모달에서 내보내기 옵션을 설정한 뒤, `Export` 버튼을 클릭합니다.
- 다운로드가 완료된 결과는 상세 페이지 내의 Results 탭에서 확인할 수 있습니다.
- **Workflow > Executions > Done** 메뉴에서 본인이 다운로드한 요청 내역을 확인할 수 있습니다.
