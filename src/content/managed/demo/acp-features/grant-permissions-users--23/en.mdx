# Overview

To maintain a high level of database access control and security, QueryPie provides versatile access control mechanisms based on roles and attributes. Only authenticated users can access databases granted through QueryPie, and SQL execution is restricted to predefined query combinations.

# Granting Access Control Privileges

<BlueState>STEP 1</BlueState> Navigate to the **Privilege** menu and click the `Create Privilege Type` button to create a new privilege.
- **Name:** Assign a unique name to identify the privilege.
- **Database Type:** Specify the database type. Options include **General** and **Redis**, depending on the query properties.
- **Privilege Type:** Define the queries allowed under this privilege.
- **Functions:** Choose the operations that can be performed under this privilege.
  - These settings control functionalities within the QueryPie Web Editor.

<BlueState>STEP 2</BlueState> Go to the **Access Control** menu and select the user or group to whom you want to grant the privilege.

<BlueState>STEP 3</BlueState> Identify the target connection and apply the relevant privilege.
